# Kenshi Online

**Мультиплеерный мод на 16 игроков для Kenshi**

[![Discord](https://img.shields.io/badge/Discord-Присоединяйтесь-5865F2?logo=discord&logoColor=white)](https://discord.gg/uNN6Pwjg)

[English version](README.md)

Kenshi Online превращает одиночную RPG с открытым миром **Kenshi** в мультиплеерную игру. Исследуйте пустоши, сражайтесь, стройте базы и торгуйте вместе с друзьями — до 16 игроков на одном сервере. Мод использует нативную систему плагинов Ogre3D, сетевую библиотеку ENet и интеграцию с MyGUI — никакой ручной инъекции DLL не требуется.

## Возможности

- **До 16 игроков** на одном выделенном сервере
- **Полная репликация сущностей** — персонажи, NPC, боевая система, здания, предметы, отряды, фракции
- **Серверная авторитетность** — боевая система и мировое состояние контролируются сервером
- **Зонная система интересов** — эффективное использование трафика (~6 КБ/с на игрока)
- **Выделенный сервер** с сохранением мира, автосохранением и консольными командами
- **Мастер-сервер** для браузера серверов (автоматическое обнаружение)
- **Нативный интерфейс MyGUI** — статус-бар, чат с временными метками, список игроков, панель отладки
- **Инъекция через Ogre-плагин** — просто запустите и играйте, никакой ручной настройки
- **13 команд клиента** — `/tp`, `/players`, `/time`, `/ping`, `/debug` и другие
- **Адаптивная интерполяция** — сплайны Эрмита, оценка джиттера, коррекция рывков
- **Автоматический UPnP** — сервер автоматически пробрасывает порты
- **Скрипт установки** — `install.bat` для быстрой установки в один клик

## Быстрый старт (Игроки)

### Вариант А: Установка в один клик
1. Скачайте последний релиз из [Releases](https://github.com/The404Studios/Kenshi-Online/releases)
2. Распакуйте папку `dist/`
3. Запустите `install.bat` — он автоматически найдёт вашу установку Kenshi (Steam или GOG)
4. Запустите Kenshi обычным способом — на главном меню появятся кнопки **HOST GAME** и **JOIN GAME**
5. Нажмите **JOIN GAME**, введите адрес сервера и играйте

### Вариант Б: Через инжектор
1. Скачайте `KenshiMP.Injector.exe` и `KenshiMP.Core.dll` из [Releases](https://github.com/The404Studios/Kenshi-Online/releases)
2. Запустите `KenshiMP.Injector.exe`
3. Укажите **путь к Kenshi**, **имя игрока** и **адрес сервера**
4. Нажмите **PLAY** — Kenshi запустится с мультиплеером

### Удаление
Запустите `uninstall.bat` из папки dist, или вручную:
1. Удалите строку `Plugin=KenshiMP.Core` из `Plugins_x64.cfg`
2. Удалите `KenshiMP.Core.dll` из папки Kenshi
3. Восстановите `Kenshi_MainMenu.layout` из папки `KenshiMP_backup/`

## Быстрый старт (Хостинг сервера)

### Запуск сервера

1. Скачайте `KenshiMP.Server.exe` из [Releases](https://github.com/The404Studios/Kenshi-Online/releases)
2. Создайте файл `server.json` (или позвольте серверу сгенерировать настройки по умолчанию):

```json
{
  "serverName": "Мой Kenshi сервер",
  "port": 27800,
  "maxPlayers": 16,
  "password": "",
  "pvpEnabled": true,
  "gameSpeed": 1.0,
  "savePath": "world.kmpsave",
  "tickRate": 20,
  "masterServer": "127.0.0.1",
  "masterPort": 27801
}
```

3. Запустите `KenshiMP.Server.exe`
4. **Пробросьте порт 27800 UDP** на вашем роутере/файрволе (UPnP пытается сделать это автоматически)
5. Игроки подключаются по вашему IP-адресу или через браузер серверов

### Возможности сервера
- **Автосохранение** каждые 60 секунд в файл `world.kmpsave`
- **Сохранение мира** — сущности, время, погода сохраняются в JSON
- **UPnP** — автоматический проброс портов при возможности
- **Связь с мастер-сервером** — регистрация в браузере серверов каждые 30 секунд
- **Корректное завершение** — Ctrl+C или команда `stop` сохраняют состояние перед выходом

## Как это работает

### Метод инъекции

Kenshi Online использует **систему плагинов Ogre3D** — тот же метод, проверенный проектом [RE_Kenshi](https://github.com/BFrizzleFoShizzle/RE_Kenshi). Инжектор добавляет `Plugin=KenshiMP.Core` в файл `Plugins_x64.cfg` Kenshi, и Ogre автоматически загружает нашу DLL при инициализации движка. **Никакой инъекции в процесс, никакой ручной загрузки DLL.**

### Что происходит при запуске

1. **Ogre загружает KenshiMP.Core.dll** как плагин при запуске движка
2. **Сканер паттернов** сканирует `kenshi_x64.exe` в памяти используя IDA-стиль байтовых паттернов с масками
3. **14 модулей хуков** устанавливают перехваты через MinHook на ключевые функции игры
4. **Сетевой клиент** подключается к серверу через ENet (UDP, 3 канала)
5. **Реестр сущностей** начинает отслеживать локальные и удалённые сущности
6. **Интеграция с MyGUI** добавляет элементы мультиплеерного интерфейса
7. **Цикл обновления** синхронизирует состояние на частоте 20 Гц

### Модель владения сущностями

- **Каждый игрок** владеет своими членами отряда — они симулируются локально и обновления отправляются на сервер
- **Сервер** владеет всеми NPC и разрешает боевые взаимодействия
- **Авторитетность может передаваться** при взаимодействиях (торговля, бой)
- **Удалённые сущности** интерполируются на каждом клиенте с буфером 100мс и сплайнами Эрмита

## Обзор архитектуры

```
KenshiMP.Injector.exe     → Модифицирует Plugins_x64.cfg, запускает Kenshi
KenshiMP.Core.dll         → Загружается Ogre как плагин, перехватывает функции игры
KenshiMP.Server.exe       → Выделенный сервер (на VPS или локально)
KenshiMP.MasterServer.exe → Реестр браузера серверов (порт 27801)
KenshiMP.Common.lib       → Общие типы, протокол, сериализация, конфиг
KenshiMP.Scanner.lib      → Сканер паттернов, обёртка MinHook
```

### Основные подсистемы

| Подсистема | Описание |
|------------|----------|
| **Сканер** | Поиск паттернов в стиле IDA с масками, разрешение RIP-относительных адресов, обнаружение vtable |
| **Менеджер хуков** | Обёртка MinHook с восстановлением трамплинов и SEH-защитой от сбоев |
| **Реестр сущностей** | Отслеживание всех сущностей (локальных + удалённых) с dirty-флагами и владением |
| **Интерполяция** | Кольцевой буфер (8 снимков), адаптивная оценка джиттера, коррекция рывков |
| **Менеджер спавна** | Захват фабричной функции игры, спавн удалённых сущностей через in-place replay |
| **Контроллер игрока** | Отслеживание удалённых персонажей, запись имён/фракций |
| **Оркестратор синхронизации** | НОВОЕ: 7-этапный конвейер (экспериментальный) для структурированных обновлений |
| **Оркестратор задач** | Пул фоновых потоков с блокировкой по кадрам |
| **Сетевой клиент** | Обёртка ENet с 3 каналами |
| **Обработчик пакетов** | 50+ обработчиков типов сообщений |
| **Мост MyGUI** | Разрешение символов и создание виджетов для нативного UI Kenshi |
| **Оверлей/HUD/Меню** | Чат, список игроков, статус-бар, кнопки главного меню, браузер серверов |

## Сетевой протокол

### Параметры подключения

| Параметр | Значение |
|----------|----------|
| **Транспорт** | UDP через ENet |
| **Порт по умолчанию** | 27800 |
| **Порт мастер-сервера** | 27801 |
| **Тик-рейт** | 20 Гц (50мс) |
| **Макс. игроков** | 16 |
| **Таймаут подключения** | 5 секунд |
| **Интервал keepalive** | 1 секунда |
| **Таймаут сессии** | 10 секунд |
| **Повтор подключения** | 6 попыток, каждые 5с (всего 30с) |

### Каналы

| Канал | Режим | Используется для |
|-------|-------|------------------|
| 0 | Надёжный упорядоченный | Подключение, спавн/деспавн сущностей, синхронизация времени, зоны, здания, чат, админ |
| 1 | Надёжный неупорядоченный | Бой, характеристики, инвентарь, отряды, фракции |
| 2 | Ненадёжный последовательный | Обновления позиций (движение) |

### Сжатие

- **Кватернион**: Кодирование «наименьшие три» (32 бита: 2 бита для наибольшего компонента, 10 бит × 3 для остальных)
- **Дельта позиции**: Float16 с порогом 0.1м
- **Дельта поворота**: Порог 0.01 радиан

```
Уровень 1: Реестр сущностей (отслеживание и владение)
           ┌──────────────────────────────────────────────────┐
           │ Локальные ↔ Удалённые сущности                   │
           │ Dirty-флаги (12 категорий) → Модель авторитетности│
           │ Автомат состояний (5 состояний) → Зонное отслеживание│
           └──────────────────────────────────────────────────┘
```

### Диапазоны ID сущностей

| Диапазон | Тип | Количество |
|----------|-----|------------|
| 1–255 | Персонажи игроков | 255 |
| 256–8191 | NPC | 7 936 |
| 8192–16383 | Здания | 8 192 |
| 16384–24575 | Контейнеры | 8 192 |
| 24576–32767 | Отряды | 8 192 |

### Состояния сущностей

| Состояние | Значение | Описание |
|-----------|----------|----------|
| Неактивна | 0 | Не используется |
| Спавнится | 1 | Зарегистрирована, ожидает игровой объект |
| Активна | 2 | Полностью синхронизирована и видима |
| Деспавнится | 3 | Удаление в процессе |
| Заморожена | 4 | Приостановлена при передаче авторитетности зоны |

### Типы авторитетности

| Тип | Значение | Описание |
|-----|----------|----------|
| Нет | 0 | Управляется сервером, нет владельца-игрока |
| Локальная | 1 | Этот клиент владеет и симулирует |
| Удалённая | 2 | Другой клиент владеет, мы интерполируем |
| Хост | 3 | Серверная/хостовая авторитетность |
| Передача | 4 | Передача авторитетности в процессе |

### Зонная система интересов

- Размер зоны: **750м × 750м**
- Радиус интересов: **3×3 зоны** вокруг каждого игрока (эффективный радиус 1 500м)
- Сервер фильтрует все рассылки по зонам — отправляет обновления только для ближайших сущностей
- Расчётный трафик: **~6 КБ/с на игрока**

## Система хуков

Kenshi Online перехватывает 14 категорий игровых функций через MinHook. Все хуки защищены SEH с трассировкой для диагностики.

| Модуль хуков | Перехваченные функции | Назначение |
|--------------|----------------------|------------|
| **entity_hooks** | CharacterCreate, CharacterDestroy | Жизненный цикл сущностей, in-place replay спавн |
| **combat_hooks** | ApplyDamage, StartAttack, Death, KO, MartialArts | Репликация боя |
| **movement_hooks** | SetPosition, MoveCommand | Синхронизация позиций |
| **game_tick_hooks** | OnGameTick, GameFrameUpdate | Основной цикл синхронизации (20 Гц) |
| **render_hooks** | Present | Рендеринг UI, резервный тик |
| **world_hooks** | ZoneLoad/Unload, BuildingPlace/Destroyed | Мировые события |
| **time_hooks** | TimeUpdate | Синхронизация игрового времени и скорости |
| **inventory_hooks** | ItemPickup, ItemDrop, ItemTransfer | Репликация инвентаря |
| **squad_hooks** | SquadCreate, SquadDestroy, SquadAddMember | Управление отрядами |
| **faction_hooks** | FactionRelation | Синхронизация отношений фракций |
| **ai_hooks** | AICreate, AIPackages | Подавление ИИ для удалённых персонажей |
| **input_hooks** | KeyboardInput | Захват клавиатуры для UI |
| **building_hooks** | BuildingPlace, Destroy, Repair | Репликация зданий |
| **save_hooks** | SaveGame, LoadGame, ImportGame | Перехват сохранения/загрузки |

## Сборка из исходников

### Требования

- **Visual Studio 2022** с рабочей нагрузкой «Разработка классических приложений на C++»
- **CMake 3.20+**
- **vcpkg** (рекомендуется) или ручная настройка библиотек

### Быстрая сборка

```bash
# Клонирование с подмодулями
git clone --recursive https://github.com/The404Studios/Kenshi-Online.git
cd Kenshi-Online

# Вариант А: Используйте скрипт настройки
setup.bat

# Вариант Б: Ручная настройка
vcpkg install enet:x64-windows minhook:x64-windows nlohmann-json:x64-windows spdlog:x64-windows
cmake -B build -G "Visual Studio 17 2022" -A x64 -DCMAKE_TOOLCHAIN_FILE=%VCPKG_ROOT%/scripts/buildsystems/vcpkg.cmake
cmake --build build --config Release
```

### Результат сборки

```
build/bin/Release/
├── KenshiMP.Core.dll           # Плагин Ogre (скопировать в папку Kenshi)

├── KenshiMP.UnitTest.exe       # Модульные тесты

```

## Устранение неполадок

### Частые проблемы

**«Plugin failed to load» / Kenshi вылетает при запуске**
- Убедитесь, что `KenshiMP.Core.dll` находится в той же папке, что и `kenshi_x64.exe`
- Проверьте, что `Plugins_x64.cfg` содержит строку `Plugin=KenshiMP.Core`
- Убедитесь, что используете **64-битную** версию Kenshi (`kenshi_x64.exe`)

**«Connection timed out» (Таймаут подключения)**
- Проверьте, что сервер запущен и доступен
- Убедитесь, что порт **27800 UDP** проброшен на файрволе/роутере сервера
- Попробуйте подключиться по IP напрямую, а не через браузер серверов

**«Pattern scan failed» / хуки не устанавливаются**
- Мод рассчитан на **Kenshi v1.0.68** (последняя версия Steam/GOG)
- Другие версии могут иметь другие адреса функций
- Другие моды, модифицирующие `kenshi_x64.exe`, могут конфликтовать

**Игроки не видны / не синхронизируются**
- Проверьте оверлей отладки (клавиша обратной кавычки) для подсчёта сущностей
- Используйте `/entities` для просмотра синхронизированных сущностей
- Используйте `/ping` для проверки качества соединения

### Логи

- **Лог клиента**: `kenshi.log` в папке Kenshi
- **Лог сервера**: Вывод в консоль (stdout)
- **Диагностика сбоев**: SEH-трассировка сообщает о последнем завершённом шаге перед падением
